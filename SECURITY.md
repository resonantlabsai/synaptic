# Security Policy

## Reporting a vulnerability
If you believe you found a security issue, please report it responsibly.

- Do **not** open a public issue with exploit details.
- Prefer a GitHub Security Advisory (if enabled on the repository).
- Otherwise, open an issue titled: **"Security report"** with minimal details and request a private channel.

## Supported versions
This project is early-stage. Security fixes will generally target the most recent release.
